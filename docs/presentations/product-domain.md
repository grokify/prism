# Product Domain

## PRISM Maturity Framework

**Requirements, Adoption, and Customer Success**

---

# Product Overview

## What We Measure

The Product domain spans **two value stream layers**:

- **Requirements** - Product ideation, specs, design clarity
- **Adoption** - User engagement, activation, retention

**Goal:** Ship the right features that customers adopt and love

---

# Product Across the Value Stream

## Layer Responsibilities

| Layer | Product Focus | Key Metrics |
|-------|---------------|-------------|
| **Requirements** | Spec clarity, design quality | Completion rate, clarity score |
| **Adoption** | User engagement, feature usage | Activation, retention, adoption |
| **Support** | Customer feedback loop | NPS, feedback volume |

---

# Requirements Layer

## Getting Specs Right

| Metric | Description | Target |
|--------|-------------|--------|
| **Spec Clarity** | % with acceptance criteria | >95% |
| **Design Coverage** | % with design review | >90% |
| **Requirement Completion** | On-time delivery rate | >85% |
| **Rework Rate** | % requiring re-spec | <10% |
| **Time to Spec** | Days from idea to spec | <5 days |

# Adoption Layer

## User Engagement Metrics

| Metric | Description | Target |
|--------|-------------|--------|
| **User Activation** | % completing onboarding | >80% |

# Product Maturity Levels

## What Each Level Means

| Level | Description | Indicators |
|-------|-------------|------------|
| **M1** | Reactive | No specs, no analytics, guessing |
| **M2** | Basic | Basic specs, some usage tracking |
| **M3** | Defined | Spec templates, analytics dashboards |
| **M4** | Managed | Data-driven decisions, experimentation |
| **M5** | Optimizing | Predictive analytics, continuous discovery |

---

# M1 → M2: Basic

## Establishing Product Foundations

**Requirements:**

- [ ] Spec template exists and used
- [ ] Basic product analytics deployed
- [ ] User activation flow defined
- [ ] Feature adoption tracked (even manually)

**Key Metrics:**

- Specs written for features
- Analytics tool deployed
- Some usage data collected

---

# M2 → M3: Defined

## Standardized Product Process

**Requirements:**

- [ ] All features have acceptance criteria
- [ ] Analytics dashboards for key metrics
- [ ] Activation funnel instrumented
- [ ] Feature flags for rollouts

**Key Metrics:**

- Spec clarity > 80%
- Activation rate tracked
- Feature adoption dashboards
- Controlled rollouts standard

---

# M3 → M4: Managed

## Data-Driven Product

**Requirements:**

- [ ] A/B testing infrastructure
- [ ] Cohort analysis automated
- [ ] Activation rate > 70%
- [ ] Feature adoption targets set

**Key Metrics:**

- Experiments per quarter > 5
- Data-driven prioritization
- Retention cohorts analyzed
- NPS tracked and improving

---

# M4 → M5: Optimizing

## Predictive Product

**Requirements:**

- [ ] Predictive churn models
- [ ] Continuous user discovery
- [ ] Personalized experiences
- [ ] Customer health scores

**Key Metrics:**

- Churn prediction accuracy > 80%
- Weekly user research
- Personalization live
- Proactive intervention
