# v0.8.0 Release Notes

**Release Date:** 2026-06-07

## Highlights

- **Unified CLI** - Single CLI integrating all PRISM ecosystem modules
- **Static Site Generator** - HTML dashboards for capability maturity visualization
- **Lit Web Components** - Interactive maturity-grid component for web visualization
- **Module Renames** - prism-intelligence → prism-maturity, prism-execution → prism-roadmap

## What's New

### Unified CLI

The `prism` CLI now integrates commands from all ecosystem modules:

```bash
# Capability stack commands (from prism-capability)
prism capability list stack.json
prism capability show stack.json sast
prism capability render stack.json --format html

# Maturity commands (from prism-maturity)
prism maturity model report model.json
prism maturity state show state.json
prism maturity plan dashboard plan.json

# Roadmap commands (from prism-roadmap)
prism roadmap okr list okrs.json
prism roadmap v2mom convert doc.json
prism roadmap mrd generate spec.json

# Ecosystem commands
prism ecosystem load --dir ./ecosystem
prism validate --config prism.yaml
prism stats --dir ./ecosystem --json
```

### Static Site Generator

Generate HTML dashboards from capability stacks with maturity overlays:

```bash
# Generate site from standard directory structure
prism site generate --stack=./security/ --output=./dist

# Multiple stacks with theme
prism site generate \
  --stack=./security/ \
  --stack=./reliability/ \
  --output=./dist \
  --theme=dark

# With layer-based aggregation
prism site generate \
  --stack=./stacks/ \
  --aggregation=min \
  --output=./dist
```

**Features:**

- Homepage with cards linking to all capability stacks
- Filterable capability grids per stack
- Capability detail pages with SLI metrics
- Maturity level badges (M1-M5) with color coding
- Metrics count indicator on capability cards
- Layer-based aggregation views (min/avg methods)
- Light and dark theme support

### Lit Web Components

New `@prism/ui` TypeScript package with Lit-based web components for interactive visualization:

```bash
# Generate site with Lit component integration
prism site generate \
  --stack=./security/ \
  --output=./dist \
  --prism-ui-js=./ui/dist/prism-ui.js
```

**maturity-grid Component:**

```html
<maturity-grid
  view="by-layer"
  theme="dark"
  show-legend
  show-view-toggle
  base-url="./capabilities/">
  <script type="application/json">
    {"layers": [...], "capabilities": [...], "maturity": {...}}
  </script>
</maturity-grid>
```

**Features:**

- Interactive capability grid with maturity overlay
- Toggle between by-layer and by-category views
- Theme-aware rendering (light/dark mode sync)
- Maturity levels M1-M5 with color coding
- Legend display with level descriptions
- Click-through to capability detail pages

**Building the UI:**

```bash
cd ui/
npm install
npm run build  # outputs dist/prism-ui.js
```

### Module Renames

The ecosystem modules have been renamed for clarity:

| Old Name | New Name | Description |
|----------|----------|-------------|
| prism-intelligence | prism-maturity | SLIs, SLOs, maturity models |
| prism-execution | prism-roadmap | OKRs, V2MOMs, roadmaps |

Config types updated accordingly:

- `IntelligenceConfig` → `MaturityConfig`
- `ExecutionConfig` → `RoadmapConfig`

## Changed

### Package Rename

The `site/` package has been renamed to `sitegen/` to avoid conflict with MkDocs output directory.

```go
import "github.com/grokify/prism/sitegen"

config := sitegen.Config{
    StackPaths: []string{"./security/"},
    OutputDir:  "./dist",
    Theme:      "dark",
}
gen := sitegen.NewGenerator(config)
gen.Generate()
```
